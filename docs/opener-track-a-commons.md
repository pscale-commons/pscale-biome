*Copy-paste this into a fresh session. It is one of two parallel tracks; the other is

---

You're picking up **Track A — the human commons** on pscale-biome (an experimental
0-9 "biome" working tree). Orient first, then act.

**Read to orient:** `CLAUDE.md` (the world rules — biome is pure 0-9; never port the
old world's `_` blocks, only re-author in 0-9); the memories `project_two_track_trajectory`,
`project_reflective_compass` (the participant layer + the A-state vision), and
`project_biome_interface`; and `docs/located-block-spec.md`. Then read the live vision
block `src/biome/constitution/reflective-compass.json` (fork A is the A-state) and the
ladder `src/biome/constitution/evolution.json`.

**Goal.** Find out whether it is genuinely useful for humans to engage the biome —
David at the interface, ideally a second human, and possibly a mobius-3 agent, all
co-present. Let *real use* drive the native (0-9) evolution of the relational ladder:
**grain** (bilateral channel) → **sed** (collective) → **pool** (shared objective) →
**SAND** (content-sharing with credit/trust) — toward open business and the **A-state
attractor** (the self-organising union of equivalents; SQ/ecosquared economics). The
economics *emerges* from sustained use; do not land it cold as a feature.

**First move — use before build.** Get co-presence working first: David at `/xstream`
(the VLS interface served by `serve.py`), a second participant (another human, or a
mobius-3 agent), each visible to the other through `/relay` (presence + live vapour)
and each carrying a `located` standing (handle + where + face). Sit in it; sense what
is useful and what is missing. *Then* build the first primitive the use actually
demands — most likely a biome-0-9 **grain** (a locked bilateral block between two
handles), then a **pool** (a shared block many contribute to).

**Live surfaces.** `/xstream` (interface), `/relay` (co-presence/vapour),
`/.well-known/biome-beach` + `/mcp` (read/write blocks via the one `spark` tool),
`marks` (the open noticeboard), `located-<handle>` (each participant's standing), the
identity membrane (`src/biome/identity-membrane.json`, handle-mode, flag-gated off by
default). This commons is a *relational* cosmology, distinct from the real-world
spatial island — run it on the existing commons beach or stand up a fresh one; sense
which fits.

**Builds on.** The participant layer just landed — `handle` + `located` + the membrane
make a human and a mobius-3 agent the **same kind of located co-inhabitant**. The
interface + relay. `evolution.json` (grain/sed/pool/SAND are its Levels 2-3). The
old-world primitives in bsp-mcp (grain, sed, pool, SAND, GRIT) are **reference only —
re-author native in 0-9, never port** (see CLAUDE.md, the two-worlds boundary).

**Discipline.** Not production; no Railway/Vercel deploy without David's explicit ask.
Build what use demands, not speculative scaffolding. Map every addition to a current in
`src/biome/constitution/biome.json` — if it doesn't map, the shell grows first.
