## Issues → v003

1. **Over-reach beyond γ.** The agent edits blocks not in γ (`capabilities`, `relationships`) — freelancing past "one minimal edit per γ-entry." Root cause: those blocks sit at **point dilation** in the reflexive current, so the agent sees only their root and assumes they're empty. The guard prevents damage, but it's wasted tokens + failed-edit noise. **Fix:** tighten the δ contract to "edit only addresses named in γ (or their `target`)"; note that a point shows the root only, not whether children exist. This both stops the over-reach and sidesteps the perception blind-spot — agency stays where it matters (composing the next current, decomposing Π), it just stops redecorating unrelated blocks.
2. **F divergence calibration — watch.** Pulse 2's haiku compare flagged `conditions:1` (from pulse 1) as diverging from `purpose:1`. Plausibly real (ρ under-realised Π's specific "close a gap AND hand forward" shape), and the agent resolved it — but if F over-flags, the agent does unnecessary work. Monitor; consider a stricter "only a genuine shape-mismatch" compare prompt.
3. **Convergence — CONFIRMED.** Pulses 3 and 4 were `γ = ∅ — rest`: ρ now mirrors Π (`purpose:1.3` is a star anchor F skips; every other purpose cell is realised in conditions and coheres), so F finds no gap and the agent rests. It converges in 3 pulses. (Two tooling bugs hid this: `digest.py` filtered out rest pulses — fixed; `heartbeat.py` waited 30 min on a settled agent without saying so — fixed to stop on settle + `--interval` for a fast research cadence.)
4. **It settled on the *starter* purpose — the real gap.** The seed Π is tiny (two branches). Vision holds the actual work (coordinate, build the network, do useful things), but nothing draws on it: at `γ = ∅` the agent just idles. Per the design (vision is the reservoir; the agent is never idle), rest should **draw the next purpose from vision**, not stop. That is the missing mechanism — and it is the hard one prior versions failed at (purpose generation with continuity).
