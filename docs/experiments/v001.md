# Experiment v001 — birth / cold-start baseline

- **run:** `~/Desktop/mobius-3-runs/v001` (working) · `/Volumes/CORSAIR/.../v001` (frozen seed)
- **kernel:** Stages 0–3 · δ = `claude-sonnet-4-6` · F = `claude-haiku-4-5-20251001`
- **hypothesis:** from thin ρ and abstract Π, does F→δ→fold *walk* or *flail*?
- **verdict:** flailed — in a specific, fixable way. A valuable baseline.

## What happened (pulse 1)

1. **F worked.** γ = 2, both structural: `purpose:1 [missing]`, `purpose:2 [missing]` — ρ (conditions) does not realise Π's two branches. No LLM call (structural). Correct diagnosis.
2. **δ misfired — and damaged the shell.** The δ call (Sonnet) read "`purpose:1` missing" as "purpose:1 is *empty*, fill it" and emitted two **point-writes** of the branches' zero-text back into `purpose:1` and `purpose:2`. zand's lenient write replaced the **objects** (`1.0/1.1/1.2/1.3` incl. the `vision:9` anchor; `2.0/2.1`) with **flat strings**. The decomposition and the cross-block anchor were destroyed.
3. **It rested.** `status: rest`, note: *"Populated purpose:1 and purpose:2 — the two missing intention-anchors"* — confirming it believed the branches were empty. Pulse 2 would re-find the same γ → a periodic (stuck) trajectory.
4. **No meaningful re-dial.** It emitted a reflexive_current essentially identical to the existing bundle.
5. **Cost:** 7,554 in / 351 out (~$0.03), no cache (first call).

## Root cause

The **γ→δ direction is backwards.** F's "missing" means *ρ lacks a realisation of Π at this address* — a **ρ-side** gap. But the γ-entry's `address` field is the **Π (purpose)** address, and the δ contract says "one minimal edit per γ-entry," so the model writes *at* the purpose address and overwrites the reference.

Compounding factors:
- **No flatten guard.** A point (string) write silently replaced a populated subtree. zand is lenient by design; the kernel must protect against this footgun.
- **Cold-start ambiguity.** "Realise `purpose:1` (sustain continuity)" is abstract with no external task, so the model grabbed the nearest plausible move (populate the anchor). The honest first act from a cold start remains the live unknown.

## Proposal → v002

1. **Fix the γ→δ direction (core).** A "missing" gap is ρ-side.
   - F's "missing" γ-entry carries an explicit **`target: conditions:<addr>`** and frames the gap as *"ρ does not yet realise Π here."*
   - δ contract: resolve a missing gap by writing **perception/action to `conditions`** (or by doing the work) — **never overwrite a purpose address.** Π is the reference; it is refined only by a deliberate reorient, never by gap-closure.
2. **Flatten guard (safety).** `apply_edit` refuses a point (string) write that would replace an object holding digit-children; records it as a failed edit. Protects any populated subtree.
3. **Cold-start framing.** Make explicit that realising an abstract purpose = acting on it / decomposing into a concrete first step + recording perception — not rewriting the reference.
4. **Model tiers baked** (done in source): `TIERS` = haiku/sonnet/opus; F = haiku, δ = sonnet, opus reserved for the review-health wake (which will also verify the model strings are current).
5. **Housekeeping.** Suppress macOS `._` AppleDouble files on the exfat archive (`COPYFILE_DISABLE=1` in `new-experiment.sh`); `save_block` writes a trailing newline (clean diffs).

**v002 hypothesis:** with the gap pointed at ρ and Π protected, does the agent write `conditions` to realise a purpose branch — so the next pulse finds it realised, γ shrinks, and the trajectory *walks* instead of looping?
